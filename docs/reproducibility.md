# Reproducibility Guide

## Environment

Run commands from the repository root with the checked-in virtual environment:

```bash
.venv/bin/python --version
.venv/bin/pytest -q finauth_audit/tests
```

## Paper package

Regenerate tables and vector figures, compile the combined anonymous-review
PDF, and build the page-faithful HTML bundle:

```bash
make -C finauth_audit paper
.venv/bin/python finauth_audit/paper/build_html.py \
  --output-dir finauth_audit/paper/html
```

The PDF contains eight pages of main text, one references page, and a combined
supplement beginning on page 10. The HTML bundle renders the same pages and
local assets. The current aggregate release is v0.6.0 and includes frozen
controlled/provenance, external order-book, and actual-model proposal evidence
without redistributing licensed Databento rows or raw provider envelopes.

## Verification

```bash
.venv/bin/python finauth_audit/paper/verify_paper.py
.venv/bin/python verify_artifact.py \
  --phase real-agent-v06 --run-tests
```

The strict artifact check verifies all earlier freezes plus the v0.6 actual-model
source checksums, split isolation, proposal schemas, legal fields, frozen
archive, one-time result registry, structural N/A semantics, generated figures
and tables, and the test suite. It also verifies that a superseded working-tree
file remains recoverable from its older content-addressed snapshot and is
covered by the later frozen surface. The release builder adds a separate
clean-room archive check.

Build and verify the allowlist release:

```bash
make -C finauth_audit release
```

This command writes `finauth_audit/dist/finauth-audit-0.6.0.tar.gz` and a
verification JSON report. The archive manifest and clean-room verifier reject
sealed/community-hidden paths, outcome files, credentials, and licensed
Databento row-level data. The v0.6 namespace is exact-allowlisted: only frozen
aggregate metrics, bootstrap bounds, subgroup diagnostics, rank inference,
source-quality summaries, historical-memory aggregates, and the public verifier
report may enter the archive.

## Paper-test boundary

The controlled/provenance, external order-book, and actual-model paper tests
were each run once after their own pre-result freeze. Re-running them is
prohibited except for a documented software defect that preserves the original
output. Community-hidden, public-test, and FinAuth-Worlds hidden outcomes are
not part of ordinary reproduction.

## Figure and table policy

Figures are generated from `finauth_audit/figures/plot_all.py` as PDF, SVG, and
PNG. Generation fails on text overlap, marker overlap, boundary violations,
legend intrusion, or fonts below 8 pt. Tables are generated by
`finauth_audit/paper/generate_tables.py`; no reported numeric result is entered
manually into the manuscript.

## Runtime reference

The one-time paper test took about 81 seconds and used about 550 MB peak
resident memory on the development machine. Full test execution in the strict
verification run completed in about six seconds after artifact checks.
